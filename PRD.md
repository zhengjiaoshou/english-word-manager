## 1. 产品概述

### 1.1 产品愿景

为英语学习者提供一个**轻量、优雅、零门槛**的个人单词管理工具。用户随时查询不认识的单词，一键收藏到个人词库，并通过发音播放、数据导出等方式巩固记忆。

### 1.2 核心问题

| 用户痛点 | 当前方案 | 我们的解法 |
|---------|---------|-----------|
| 查词后容易忘记，缺乏系统记录 | 临时写在纸上 / 截屏 | 查词 → 一键收藏 → 个人词库自动管理 |
| 单词应用分散在不同工具中 | 有道词典 + 笔记本 + Excel | 统一入口：查词 + 收藏 + 导出 |
| 现有工具要么太重要么太简陋 | Anki（学习曲线陡）/ Excel（无查询） | 毛玻璃极简设计，开箱即用 |
| 没有发音辅助 | 手动搜发音网站 | 内置 TTS 英美双发音 |

### 1.3 目标用户

- **主要用户**：英语学习者（学生、职场人士），每天有 3-10 个新词学习需求
- **次要用户**：需要快速查词但不一定要收藏的轻量用户

### 1.4 产品定位

> **英语单词管家 = 查词工具 + 个人词库 + 轻量学习助手**  
> 一句话：**查得到、存得住、带得走、听得出**

### 2.2 Non-goals（本版本明确不做）

- ❌ 用户账户系统 / 云端同步 — 纯本地存储，无需登录
- ❌ 拼写检查 / 自动纠错 — 依赖 API 返回，不自己判断
- ❌ 复习算法（如间隔重复）— 目前是纯管理工具，非学习系统
- ❌ 多语言词典查询 — 仅支持英语
- ❌ 单词编辑功能 — 当前仅支持添加/删除，不可修改已有词条
- ❌ 离线词典 — 依赖 Free Dictionary API，需网络

## 3. 用户故事

### 3.1 查词流程

> **作为** 英语学习者，  
> **我希望** 在输入框中输入一个英语单词并点击查询，  
> **以便** 立即看到该单词的音标、释义和例句。

**验收标准**：
- 输入单词后按回车或点击"查询"按钮触发搜索
- 查词中显示"查询中..."加载态
- 成功时展示：单词标题 + 英式/美式音标 + 释义（按词性优先级排序，最多2条）+ 例句（最多2条）
- 未找到时显示"未找到该单词，请检查拼写"
- 网络错误时显示具体错误提示（含本地服务器启动指引）

### 3.2 收藏单词

> **作为** 英语学习者，  
> **我希望** 在查询到一个新单词后点击"添加到词库"，  
> **以便** 将这个词永久保存到我的个人词库中。

**验收标准**：
- 点击"添加到词库"后单词存入 IndexedDB
- 已存在的单词显示"已添加"且按钮禁用
- 添加成功后弹出 Toast 通知，并播放该单词发音
- 首页"今日单词"和"最近添加"自动刷新

### 3.3 词库管理

> **作为** 英语学习者，  
> **我希望** 在我的词库页面查看所有已收藏的单词，支持搜索，  
> **以便** 快速找到某个特定单词的详细信息。

**验收标准**：
- 词库以表格形式展示：#、单词、音标、释义（首条截取40字）、日期、发音按钮
- 支持输入关键词搜索（匹配单词、释义、音标）
- 点击行弹出单词详情弹窗
- 弹窗内可查看完整信息并支持删除

### 3.4 发音辅助

> **作为** 英语学习者，  
> **我希望** 听到单词的标准发音，  
> **以便** 纠正我的发音并加深记忆。

**验收标准**：
- 搜索结果中提供 ▶️ 单次播放按钮
- 提供 🔊 英美双发音按钮（英式语速 0.85，美式语速 0.9）
- 优先使用 Microsoft Jenny (en-US) 语音引擎
- 列表/表格中每个单词旁有小播放按钮
- 播放时自动取消上一个正在播放的语音

### 3.5 数据导入导出

> **作为** 英语学习者，  
> **我希望** 将词库导出为文件备份，并能从备份文件恢复，  
> **以便** 在不同设备间迁移数据或分享词库。

**验收标准**：
- JSON 导出：包含元数据（导出时间、总数、版本）+ 单词列表
- Markdown 导出：按序号排列，含音标、释义、例句、记录日期
- JSON 导入：解析文件 → 去重 → 统计导入/跳过数量 → Toast 反馈
- 文件下载通过浏览器 Blob + 临时 `<a>` 标签实现

### 3.6 视觉体验

> **作为** 用户，  
> **我希望** 使用一个界面美观、操作流畅的工具，  
> **以便** 在学习过程中保持愉悦的心情。

**验收标准**：
- 毛玻璃（Glassmorphism）风格：半透明背景 + backdrop-filter 模糊 + 柔和阴影
- 紫蓝渐变背景 + 三个浮动装饰圆形（持续 20s 循环动画）
- Sticky 导航栏，毛玻璃材质
- 支持暗色/亮色一键切换，偏好存入 localStorage
- 移动端自适应（768px / 480px 断点）
- 自定义滚动条样式
- 页面切换淡入动画（0.35s）

## 4. 技术架构

```
┌─────────────────────────────────────────┐
│              Presentation Layer          │
│   index.html  │  style.css  │  ui.js    │
├─────────────────────────────────────────┤
│             Application Layer            │
│       app.js (主控制器 + 事件绑定)        │
├──────────────────┬──────────────────────┤
│   Data Layer     │   Service Layer       │
│   db.js          │   dictionary.js       │
│   (IndexedDB)    │   (Free Dict API)     │
│                  │   tts.js (Web Speech) │
└──────────────────┴──────────────────────┘
```

### 4.1 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端框架 | 原生 HTML/CSS/JS | 零依赖、零构建、极简部署 |
| 数据存储 | IndexedDB | 浏览器原生、支持大数据量、按索引查询 |
| 词典 API | Free Dictionary API | 免费、无需 Key、支持音标和例句 |
| 发音 | Web Speech API | 浏览器原生、无需额外请求 |
| 辅助工具 | Python (helpers.py) | CLI 查词、Markdown 卡片生成、PDF 导出 |

### 4.2 数据模型

```typescript
// IndexedDB 单词记录
interface WordRecord {
  id: number;           // 自增主键
  word: string;         // 单词（小写）
  phoneticUk: string;   // 英式音标
  phoneticUs: string;   // 美式音标
  definitions: string[];// 释义列表
  examples: { en: string; cn?: string }[]; // 例句
  audioData: string;    // 音频数据（预留）
  audioFormat: string;  // 音频格式（预留）
  addedAt: string;      // ISO 8601 时间戳
  source: string;       // 数据来源 'dictionary-api' | 'import'
}

// 导出 JSON 结构
interface ExportData {
  metadata: {
    exportedAt: string;
    totalWords: number;
    version: string;
  };
  words: Omit<WordRecord, 'audioData' | 'audioFormat'>[];
}
```

## 6. 指标体系

### 6.1 北极星指标

| 指标 | 定义 | 当前基线的推测 | 目标（3个月） |
|------|------|--------------|-------------|
| **DAU (日活用户)** | 每天至少执行一次查询或浏览词库的用户数 | 待埋点 | ≥ 50 |
| **单词收藏率** | 查词后点击"添加到词库"的比例 | 待埋点 | ≥ 40% |
| **词库留存率** | 添加第1个单词后，7天内再次打开的用户比例 | 待埋点 | ≥ 30% |

### 6.2 驱动指标

| 指标 | 说明 |
|------|------|
| 日均查词次数 | 反映用户活跃度 |
| 人均词库单词数 | 反映收藏行为深度 |
| 导出次数 / 人 | 反映数据价值感知 |
| 发音播放次数 | 反映发音功能使用频率 |

### 6.3 健康指标

| 指标 | 说明 | 警戒线 |
|------|------|--------|
| API 查询成功率 | Free Dictionary API 可用性 | < 95% |
| IndexedDB 读写错误率 | 存储可用性 | > 0.1% |
| 页面加载时间 | 首屏渲染 | > 3s |
| 移动端崩溃率 | JS 错误率 | > 1% |

> ⚠️ **当前状态**：v1.0 尚未接入埋点系统，以上指标为设计层面的定义。建议在 v1.1 中接入简单的埋点（如 Google Analytics / 自建 Beacon）。

## 7. 风险与假设

### 7.1 技术风险

| 风险 | 严重度 | 缓解措施 |
|------|--------|---------|
| Free Dictionary API 不稳定或下线 | 高 | 调研备用 API（如 Merriam-Webster、Oxford）；考虑 v2.0 自建离线词库 |
| IndexedDB 在部分浏览器/隐私模式下不可用 | 中 | 优雅降级：检测不支持时提示用户 |
| Web Speech API 不同浏览器发音质量不一致 | 中 | 优先推荐 Chrome/Edge 浏览器 |
| 数据丢失（浏览器清除缓存） | 中 | 定期导出提醒；后续版本可加云同步 |

### 7.2 产品假设

| 假设 | 验证方式 |
|------|---------|
| 用户愿意手动查词并收藏（非自动推送） | 观察 DAU / 收藏率 |
| 用户对"毛玻璃"设计风格接受度好 | 用户反馈收集 |
| 英美双发音是差异化价值 | 发音功能使用率 |
| 导出 Markdown 有实际使用场景 | 导出功能点击率 |
